# Deployment Notes — Hopf Reservoir CNN on Arm Cortex-M

## Overview

This CNN was designed so every inference layer maps to a CMSIS-NN kernel
function. The TFLite INT8 model generated by `pipeline/convert.py` can be
deployed via TFLite Micro or via the raw CMSIS-NN parameter header
(`cmsis_nn_params.h`).

Use the target-core guidance conservatively:
- `M4` is the lowest-cost tier and should prefer classical, tree-based,
  nearest-neighbour, and prototype models.
- `M33` adds some headroom for lightweight learned models such as the
  depthwise CNN and ridge readout.
- `M55` is the first core in this package where the heavier CNN/sequence
  families and the current denoiser are part of the official support story.
- `M85` is the premium tier for the heaviest sequence and representation
  learning models.

---

## Cortex-M4 (e.g. STM32F4, nRF52-class DSP tier)

### Compiler Flags

```
-mcpu=cortex-m4 -mfpu=fpv4-sp-d16 -mfloat-abi=hard -O2
```

### CMSIS-NN DSP Expectations

CMSIS-NN uses the Cortex-M4 DSP extension for the optimized integer kernels.
Verify that the compiler sees `__ARM_FEATURE_DSP` or `ARM_MATH_DSP` when the
CMSIS-NN sources are built.

### Recommended Model Classes

- Classical signal models
- Tree-based models
- KNN / prototype-style models
- Echo-state style readouts

### Not in the Official M4 Profile

- Heavy CNN families (`A-E`)
- Heavy sequence models (`V`, `W`)
- Current TCN denoiser

The practical issue is not only conversion. `M4` deployment is constrained by
arena size, scratch buffers, preprocessing memory, and latency headroom.

### Memory Budget

- Treat `48 KB` as the conservative effective inference budget for the package
- Use board-specific validation before claiming support beyond the lightweight
  model set

## Cortex-M33 (e.g. STM32U5, nRF5340)

### Compiler Flags

```
-mcpu=cortex-m33 -mfpu=fpv5-sp-d16 -mfloat-abi=hard -O2
```

### CMSIS-NN Linkage

Link the INT8-specific source files, NOT the generic fallbacks:

```
CMSIS/NN/Source/ConvolutionFunctions/arm_convolve_s8.c
CMSIS/NN/Source/PoolingFunctions/arm_max_pool_s8.c
CMSIS/NN/Source/FullyConnectedFunctions/arm_fully_connected_s8.c
CMSIS/NN/Source/SoftmaxFunctions/arm_softmax_s8.c
CMSIS/NN/Source/ActivationFunctions/arm_relu_s8.c
CMSIS/NN/Source/BasicMathFunctions/arm_elementwise_add_s8.c
```

### DSP SIMD

`arm_convolve_s8()` uses DSP SIMD instructions (SMLAD) on M33.
Verify with `-mcpu=cortex-m33` — the compiler must see `__ARM_FEATURE_DSP`.

### ADC DMA Configuration

```c
// Double-buffered DMA at 100 kHz for the Hopf oscillator x(t) output
HAL_ADC_Start_DMA(&hadc1, (uint32_t *)adc_buffer, ADC_BUF_SIZE);
// Configure TIM trigger at 100 kHz: PSC=0, ARR=(SystemCoreClock/100000)-1
// 16-bit resolution, single channel
```

### Scratch Buffer

```c
// Allocate at init — use the largest across all conv layers
int32_t scratch_size = arm_convolve_s8_get_buffer_size(
    &conv1_input_dims, &conv1_filter_dims);
int8_t *scratch_buf = (int8_t *)malloc(scratch_size);
```

### Memory Budget

- **Max model**: 32 KB weights + 32 KB activations = 64 KB effective inference
  budget
- Feature map preprocessing (atanh, reshape) runs before inference in the
  same 64 KB, so schedule it to complete before the inference buffer is needed.

## Cortex-M55 (e.g. Corstone-300, AN547)

### Compiler Flags

```
-mcpu=cortex-m55 -mfpu=fpv5-d16+mvei -mfloat-abi=hard -O2
```

### Helium MVE Acceleration

CMSIS-NN auto-detects Helium (M-Profile Vector Extension) on M55.
`arm_convolve_s8()` uses `vmlaq_s8` intrinsics for 4x throughput over M33.
No code changes needed — the same CMSIS-NN source compiles differently.

### Vela Compiler

Run the TFLite model through Arm Vela for Ethos-U offloading:

```bash
vela model.tflite
```

### Feature Map Preprocessing — MVE Vectorisation

Hand-vectorise the atanh + reshape preprocessing with `arm_mve.h`:

```c
#include <arm_mve.h>

// Vectorised offset subtraction (part of quantisation scaling)
int8x16_t v = vldrbq_s8(src);
vstrbq_s8(dst, vqsubq_s8(v, offset));
```

### QEMU Emulation

```bash
# Cycle-accurate validation on MPS3 AN547 (Corstone-300)
qemu-system-arm -M mps3-an547 -kernel firmware.elf -nographic
```

## Cortex-M85 + Ethos-U55 NPU (e.g. Corstone-310)

### Vela Compilation

```bash
vela model.tflite \
    --accelerator-config=ethos-u55-256 \
    --optimise Performance
```

### Vela Report Verification

Check the Vela report output — **all Conv2D and Dense layers must show
"ethos-u" placement, not "cpu"**:

```
Layer    | Placement | Cycles
---------|-----------|-------
conv1    | ethos-u   | 1234
conv2    | ethos-u   | 1234
dense1   | ethos-u   | 567
output   | ethos-u   | 89
```

Any layer falling back to CPU negates the NPU advantage. Fix by removing
unsupported ops (batch norm, non-standard activations). This model is
designed to have zero CPU fallback layers.

### Dual-Core Architecture

```
M33 secondary core (low power, 1.8V):
  - Runs ADC DMA at 100 kHz (HAL_ADC_Start_DMA)
  - Skip-samples to 4 kHz
  - Builds 200x100 uint8 feature map in shared SRAM
  - Triggers interrupt when feature map buffer is full
  - Never needs to know about the CNN model

M85 primary core (high performance):
  - Sleeps until M33 interrupt arrives
  - Wakes, runs Vela-compiled model via Ethos-U55 NPU
  - NPU handles all Conv2D/Dense layers in hardware
  - Writes classification result to shared register
  - Returns to sleep
```

### FVP Validation

```bash
# Fixed Virtual Platform simulation
FVP_Corstone_SSE-300_Ethos-U55 \
    -a cpu0*=inference.elf \
    --stat
```

### Performance Notes

- Ethos-U55-256 config: 256 MACs/cycle
- Expected inference time for this model: < 5 ms at 400 MHz
- Power: ~50 mW during NPU inference burst

## Common Notes

### Input Format

All targets expect:
- Input shape: `(1, 200, 100, 1)` — batch=1, 200 time steps, 100 virtual nodes, 1 channel
- Input type: `int8` (range [-128, 127])
- The feature extraction pipeline outputs `uint8` [0, 255]; subtract 128 to get `int8`
  (this offset is encoded in the TFLite model's input zero-point)

### CMSIS-NN Kernel Map

| Model Layer         | CMSIS-NN Kernel                   | Source File                                          |
|---------------------|-----------------------------------|------------------------------------------------------|
| Conv2D + ReLU       | `arm_convolve_s8()` + fused ReLU  | ConvolutionFunctions/arm_convolve_s8.c               |
| Conv2D 1×1          | `arm_convolve_1x1_s8()`           | ConvolutionFunctions/arm_convolve_1x1_s8.c           |
| DepthwiseConv2D     | `arm_depthwise_conv_s8()`         | ConvolutionFunctions/arm_depthwise_conv_s8.c         |
| MaxPool2D           | `arm_max_pool_s8()`               | PoolingFunctions/arm_max_pool_s8.c                   |
| Flatten             | `arm_reshape_s8()` (no-op)        | —                                                    |
| Dense + ReLU        | `arm_fully_connected_s8()`        | FullyConnectedFunctions/arm_fully_connected_s8.c     |
| Concatenate         | `arm_concatenation_s8()`          | ConcatenationFunctions/arm_concatenation_s8.c        |
| Softmax             | `arm_softmax_s8()`                | SoftmaxFunctions/arm_softmax_s8.c                    |

## Multi-Model Deployment (x(t) + y(t) architectures)

The extended model suite processes both Hopf oscillator states x(t) and y(t).
Five input representations are supported:

| Representation | Description                          | Shape          |
|----------------|--------------------------------------|----------------|
| `x_only`       | x(t) feature map (paper 2 baseline)  | (200, 100, 1)  |
| `y_only`       | y(t) feature map (same pipeline)     | (200, 100, 1)  |
| `xy_dual`      | x and y stacked as channels          | (200, 100, 2)  |
| `phase`        | r(t) = sqrt(x²+y²) orbit radius      | (200, 100, 1)  |
| `angle`        | θ(t) = arctan2(y,x) phase angle      | (200, 100, 1)  |

## Model E — Late Fusion Dual-Input DMA

Model E (`cnn_xy_fusion`) has two separate input tensors, one for x(t) and
one for y(t). Each input requires its own DMA-filled buffer.

### Single ADC — M33 (time-multiplexed)

```c
// x(t) window: sample 1 second of x(t) output
HAL_ADC_Start_DMA(&hadc1, (uint32_t *)buffer_x, 200*100);

// y(t) window: route to y ADC input, sample next 1 second
// Hardware switch: multiplexer toggles Hopf output channel
HAL_ADC_Stop_DMA(&hadc1);
HAL_GPIO_WritePin(GPIOA, MUX_SEL_PIN, GPIO_PIN_SET);  // select y(t) output
HAL_ADC_Start_DMA(&hadc1, (uint32_t *)buffer_y, 200*100);
```

### Dual ADC — M85 (simultaneous sampling)

```c
// Configure ADC1 for x(t), ADC2 for y(t)
// Both sampled simultaneously via dual interleaved mode

// ADC1 — x(t) channel
hadc1.Init.ScanConvMode = DISABLE;
HAL_ADC_Start_DMA(&hadc1, (uint32_t *)buffer_x, ADC_BUF_SIZE);

// ADC2 — y(t) channel
hadc2.Init.ScanConvMode = DISABLE;
HAL_ADC_Start_DMA(&hadc2, (uint32_t *)buffer_y, ADC_BUF_SIZE);

// TIM3 triggers both ADCs simultaneously at 100 kHz
// PSC=0, ARR=(SystemCoreClock/100000)-1
// ADC1_ETREG and ADC2_ETREG both set to TIM3_TRGO
```

### TFLite Micro inference with two inputs

```c
// After both DMA buffers are filled and feature extraction complete:
TfLiteTensor* input_x = interpreter->input(0);  // x branch
TfLiteTensor* input_y = interpreter->input(1);  // y branch

// Copy quantised feature maps
memcpy(input_x->data.int8, feature_x_int8, 200*100*sizeof(int8_t));
memcpy(input_y->data.int8, feature_y_int8, 200*100*sizeof(int8_t));

interpreter->Invoke();

TfLiteTensor* output = interpreter->output(0);
int8_t predicted_class = std::max_element(
    output->data.int8,
    output->data.int8 + n_classes
) - output->data.int8;
```

## Model F — Depthwise Separable CNN

Model F (`depthwise_cnn`) uses `arm_depthwise_conv_s8()` for 8-9× fewer MACs.
Link the additional depthwise source:

```
CMSIS/NN/Source/ConvolutionFunctions/arm_depthwise_conv_s8.c
CMSIS/NN/Source/ConvolutionFunctions/arm_convolve_1x1_s8.c
```

On M55 with Helium, `arm_depthwise_conv_s8()` uses `vmlaq_s8` with 4×
throughput. Expected inference time on M55 at 160 MHz: **< 2 ms**.
